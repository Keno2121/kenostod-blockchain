📝 What Are Blockchain Transactions?

A blockchain transaction is a record of value transfer between two parties on the blockchain network. Think of it like a digital check that gets permanently recorded in a public ledger that everyone can verify.

Components of a Transaction

Every blockchain transaction contains several key pieces of information:

  • Sender Address: The wallet address sending the funds (derived from the sender's public key)
  • Recipient Address: The wallet address receiving the funds
  • Amount: How much cryptocurrency is being transferred
  • Digital Signature: Cryptographic proof that the sender authorized this transaction
  • Transaction Fee: A small payment to the network for processing

💡 Key Concept: Immutability

Once a transaction is confirmed on the blockchain, it cannot be altered or deleted. This is what makes blockchain so secure and trustworthy - there's no "undo" button (except in Kenostod's innovative 5-minute reversal window, which you'll learn about in Course 3!).

🔗 The UTXO Model

Kenostod uses the UTXO (Unspent Transaction Output) model, the same system used by Bitcoin. This is fundamentally different from the account-based model used by traditional banks or Ethereum.

How UTXO Works

Imagine you have physical bills and coins. When you pay for something, you can't tear a $20 bill in half. Instead:

  • You spend entire "coins": Each UTXO is like a digital coin that must be spent in full
  • You receive change: If your UTXO is larger than needed, the extra is sent back to you as a new UTXO
  • UTXOs are consumed: Once spent, a UTXO cannot be used again

Figure: How transactions flow from creation through signing to blockchain confirmation

UTXO Example

Let's say you have 100 KENO and want to send 30 KENO to a friend:

Your Wallet: 1 UTXO worth 100 KENO Transaction: ├── Input: Your 100 KENO UTXO (consumed) ├── Output 1: 30 KENO → Friend's wallet ├── Output 2: 69.9 KENO → Back to you (change) └── Fee: 0.1 KENO → Network miners Result: Your old UTXO is gone Friend has 1 new UTXO (30 KENO) You have 1 new UTXO (69.9 KENO)

✅ Benefits of UTXO

The UTXO model provides excellent privacy (you can use new addresses for change), enables parallel transaction validation, and makes it impossible to double-spend the same funds.

✍️ Digital Signatures

A digital signature is cryptographic proof that you authorized a transaction. It's like a handwritten signature, but mathematically impossible to forge.

How Digital Signatures Work

  • Create Transaction Data: Combine sender, recipient, amount, and other details
  • Hash the Data: Create a unique fingerprint of the transaction using SHA-256
  • Sign with Private Key: Use your private key to encrypt the hash, creating the signature
  • Anyone Can Verify: Others use your public key to verify the signature matches the transaction

💡 Why This Matters

Digital signatures prove three things simultaneously:

  • Authentication: The transaction came from the owner of the private key
  • Integrity: The transaction wasn't modified after signing
  • Non-repudiation: The sender can't deny they signed it

The Signing Process

// Simplified transaction signing process const transaction = { from: "your_wallet_address", to: "recipient_address", amount: 50, timestamp: Date.now() }; // 1. Create message hash const message = JSON.stringify(transaction); const messageHash = SHA256(message); // 2. Sign with private key (using secp256k1) const signature = privateKey.sign(messageHash); // 3. Attach signature to transaction transaction.signature = signature;

⚠️ Security Warning

Your private key should NEVER leave your device. Signing happens locally, and only the signature (not your private key) is sent to the network.

✅ Transaction Validation

Before a transaction is added to the blockchain, it goes through rigorous validation. This ensures only legitimate transactions are recorded.

Validation Steps

  • Signature Verification: Confirm the digital signature is valid and matches the sender's public key
  • Balance Check: Verify the sender has sufficient UTXOs to cover the amount plus fees
  • Double-Spend Check: Ensure the UTXOs haven't already been spent in another transaction
  • Format Validation: Check that all required fields are present and properly formatted
  • Fee Verification: Confirm the transaction fee meets the minimum requirement

The Mempool

Valid transactions enter the mempool (memory pool), a waiting area for unconfirmed transactions. Miners select transactions from the mempool to include in the next block, typically prioritizing those with higher fees.

💡 Transaction States

  • Pending: In the mempool, waiting to be mined
  • Confirmed: Included in a block on the blockchain
  • Failed: Rejected due to validation errors

💰 Transaction Fees

Transaction fees are small payments made to the network for processing your transaction. They serve multiple purposes:

Why Fees Exist

  • Prevent Spam: Makes it expensive to flood the network with worthless transactions
  • Reward Miners: Incentivizes miners to include your transaction in a block
  • Prioritization: Higher fees = faster confirmation during congestion
